## 5. ページ別詳細仕様

### 5.1 トップページ (/)

#### セクション構成

```
1. ヒーローセクション
   - メインビジュアル（店舗写真）
   - キャッチコピー：「一人一人の男性に「光」と「印象」を…」
   - CTAボタン：「ご予約はこちら」

2. コンセプト紹介（簡易版）
   - 短いコンセプト文
   - 「詳しく見る」→ 店舗情報ページへリンク

3. メニュー紹介（抜粋）
   - 主要メニュー3〜4件
   - 「メニューを見る」→ メニューページへリンク

4. お知らせ（最新3件程度）
   - タイトル + 日付
   - 「一覧を見る」→ お知らせページへリンク

5. アクセス概要
   - 住所
   - 営業時間
   - Googleマップ埋め込み（小）

6. 予約導線
   - CTAボタン：「Web予約する」

7. フッター
   - ロゴ
   - ナビゲーションリンク
   - SNSリンク
   - コピーライト
```

### 5.5 ご予約ページ (/reservation)

#### 予約システム機能

```
1. カレンダー表示
   - 月表示
   - 空き状況表示（○△×）

2. 予約フロー
   Step 1: 日付選択
   Step 2: 時間選択
   Step 3: メニュー選択
   Step 4: お客様情報入力
   Step 5: 確認
   Step 6: 完了

3. 予約情報入力項目
   - お名前（必須）
   - フリガナ（必須）
   - 電話番号（必須）
   - メールアドレス（必須）
   - ご要望・備考（任意）
```

## 7. 技術要件

### 7.1 推奨技術スタック

| レイヤー | 技術 |
|----------|------|
| フロントエンド | Next.js / React |
| スタイリング | Tailwind CSS |
| CMS（お知らせ） | Headless CMS or 自作管理画面 |
| 予約システム | 自作（Firebase等） |
| ホスティング | Vercel |
| データベース | PostgreSQL or Firebase |
| メール送信 | SendGrid / Resend |

### 7.2 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンシブ | スマホ・タブレット・PC対応 |
| 表示速度 | PageSpeed 90点以上目標 |
| SEO | meta設定、構造化データ対応 |
| MEO | Googleビジネスプロフィール連携 |
| SSL | 必須（HTTPS） |

### 7.3 MEO対応

- Googleビジネスプロフィール初期設定サポート
- 構造化データ設定（LocalBusiness）
- NAP情報の統一（店名・住所・電話番号）
